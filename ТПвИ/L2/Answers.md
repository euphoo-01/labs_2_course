# Ответы на вопросы к Лабораторной работе №2

**15. Поясните назначение паттерна Builder. Укажите: в каком месте ASPA применяется этот паттерн.**

**Паттерн Builder (Строитель)** — это порождающий паттерн проектирования, который позволяет создавать сложные объекты пошагово. Строитель даёт возможность использовать один и тот же код строительства для получения разных представлений объектов.

**В ASP.NET Core (ASPA):** Паттерн применяется при инициализации приложения в файле `Program.cs`.
```csharp
var builder = WebApplication.CreateBuilder(args);
// ... конфигурация сервисов ...
var app = builder.Build();
```
Объект `WebApplicationBuilder` позволяет настроить сервисы, конфигурацию, логирование и хост перед тем, как создать сам объект приложения `WebApplication` методом `Build()`.

---

**16. Поясните назначение паттерна Chain of Responsibility. Укажите: в каком месте ASPA применяется этот паттерн.**

**Паттерн Chain of Responsibility (Цепочка обязанностей)** — это поведенческий паттерн проектирования, который позволяет передавать запросы последовательно по цепочке обработчиков. Каждый последующий обработчик решает, может ли он обработать запрос сам и стоит ли передавать запрос дальше по цепи.

**В ASP.NET Core (ASPA):** Паттерн реализуется через конвейер обработки запросов (Request Pipeline) с использованием **Middleware**.
```csharp
app.UseWelcomePage("/aspnetcore");
app.UseStaticFiles();
app.MapGet("/", ...);
```
Каждый компонент Middleware может либо обработать запрос и вернуть ответ (прервав цепочку), либо выполнить какую-то логику и передать контекст запроса следующему компоненту (`next()`).

---

**17. Поясните понятие: Middleware.**

**Middleware (Промежуточное ПО)** — это программный код, который встраивается в конвейер обработки приложений для обработки запросов и ответов.
Каждый компонент:
1. Выбирает, передавать ли запрос следующему компоненту в конвейере.
2. Может выполнять работу до и после вызова следующего компонента в конвейере.

В ASP.NET Core middleware используются для таких задач, как аутентификация, обработка ошибок, обслуживание статических файлов, маршрутизация и т.д.

---

**18. Поясните понятие: статический файл.**

**Статический файл** — это файл, который хранится на сервере и отправляется клиенту (браузеру) "как есть", без дополнительной обработки на стороне сервера.
Примеры: HTML-файлы, CSS-стили, JavaScript-скрипты, изображения (.jpg, .png), шрифты.

---

**19. Какой отличительный (общепринятый) признак наименования, подключаемого Middleware-компонента.**

Общепринятым признаком является использование префикса **Use** в названии метода расширения для `IApplicationBuilder` (или `WebApplication`).
Примеры: `UseStaticFiles`, `UseRouting`, `UseEndpoints`, `UseAuthentication`.

---

**20. Перечислите известные вам встроенные Middleware-компоненты.**

Некоторые встроенные компоненты:
1. `UseStaticFiles` — возвращает статические файлы.
2. `UseDefaultFiles` — перезаписывает URL для корневого каталога на имя файла по умолчанию (например, index.html).
3. `UseDirectoryBrowser` — позволяет просматривать содержимое каталогов.
4. `UseWelcomePage` — отображает встроенную приветственную страницу.
5. `UseExceptionHandler` — перехватывает исключения и отправляет пользователю страницу с ошибкой.
6. `UseDeveloperExceptionPage` — отображает подробную информацию об ошибках (для разработки).
7. `UseRouting` — сопоставляет запросы с конечными точками.
8. `UseCors` — настраивает общий доступ к ресурсам (CORS).
9. `UseAuthentication` — аутентификация пользователя.
10. `UseAuthorization` — авторизация пользователя.
11. `UseSession` — поддержка сессий.

---

**21. Какое имя директория ASPA (по умолчанию), применяемого для хранения статических файлов приложения.**

По умолчанию используется директория **`wwwroot`**.

**22. ♣ Кто такой Джон фон Нейман? ☺**

**Джон фон Нейман** (John von Neumann) — венгро-американский математик, физик, компьютерный учёный и эрудит.
Он внёс фундаментальный вклад во многие области, включая:
*   **Архитектуру компьютеров:** Архитектура фон Неймана (принцип хранения программы и данных в одной памяти) лежит в основе большинства современных компьютеров.
*   **Квантовую механику:** Математические основы квантовой механики.
*   **Теорию игр:** Один из создателей теории игр.
*   **Ядерную физику:** Участник Манхэттенского проекта.
*   **Клеточные автоматы:** Создатель теории самовоспроизводящихся автоматов.

Его фотография используется в лабораторной работе в качестве статического ресурса.
