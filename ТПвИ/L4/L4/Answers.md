## Задание 3

**11. Поясните назначение подчеркнутых или обведенных фрагментов программного кода:**
* `app.UseExceptionHandler("/Celebrities/Error");` — промежуточное ПО (middleware), которое перехватывает необработанные исключения в приложении и перенаправляет выполнение запроса на указанный путь (`/Celebrities/Error`) для централизованной обработки ошибки и формирования ответа клиенту.
* `app.MapFallback(...)` — задает обработчик маршрута по умолчанию (fallback). Он вызывается, если ни один из зарегистрированных маршрутов (endpoint) не совпал с запрошенным URL. В данном случае возвращает статус `404 Not Found` с сообщением о неподдерживаемом пути.
* `IExceptionHandlerFeature` — интерфейс, предоставляемый ASP.NET Core в контексте обработки ошибок. Он позволяет получить объект исключения (`Error`), которое привело к вызову обработчика ошибок, для его анализа (типа ошибки, сообщения) и логирования.
* `Results.Problem(...)` — вспомогательный метод, который формирует стандартизированный ответ с описанием ошибки (в формате `ProblemDetails` согласно RFC 7807). По умолчанию возвращает HTTP-статус `500 Internal Server Error`, включает заголовок (`title`), детали (`detail`) и информацию об экземпляре (`instance`).
* `if (ex is FoundByIdException) rc = Results.NotFound(ex.Message);` — проверка типа перехваченного исключения. Если это `FoundByIdException` (пользовательское исключение, которое выбрасывается, если сущность не найдена в репозитории), формируется HTTP-ответ со статусом `404 Not Found` и сообщением об ошибке.

**12. Поясните фрагменты кода, участвовавшие в обработке каждого запроса, и статус ответа:**
1. Запрос списка всех записей (GET `/Celebrities`):
   * Фрагмент кода: `app.MapGet("/Celebrities", () => repository.getAllCelebrities());`
   * Статус: `200 OK`. Статус формируется автоматически при успешном возврате результата функции.
2. Запрос существующей записи (GET `/Celebrities/{id}`):
   * Фрагмент кода: `app.MapGet("/Celebrities/{id:int}", (int id) => { ... });`
   * Статус: `200 OK`. Формируется автоматически при успешном возврате объекта `Celebrity`.
3. Запрос несуществующей записи (GET `/Celebrities/{несуществующий_id}`):
   * Фрагмент кода: Сначала `app.MapGet` (где срабатывает `throw new FoundByIdException`), затем `app.UseExceptionHandler` перенаправляет запрос в `app.Map("/Celebrities/Error", ...)`, где исключение обрабатывается в `if (ex is FoundByIdException)`.
   * Статус: `404 Not Found`. Формируется явно вызовом `Results.NotFound()`.
4. Запрос с некорректным ID (GET `/Celebrities/не_число`):
   * Фрагмент кода: Так как тип ID в маршруте ограничен `int` (`{id:int}`), маршрут не совпадает. Срабатывает `app.MapFallback()`.
   * Статус: `404 Not Found`. Формируется явно вызовом `Results.NotFound(new {error = ...})`.
5. Добавление записи (POST `/Celebrities`):
   * Фрагмент кода: `app.MapPost("/Celebrities", (Celebrity celebrity) => { ... });`
   * Статус: `200 OK`. Формируется автоматически при успешном возврате нового объекта.

**21. Предложите изменение в программном коде, чтобы в поле detail была более подробная информация об ошибке:**
Для получения более подробной информации необходимо извлечь сообщение о корневой ошибке, которое часто кроется во вложенном исключении (`InnerException`). Измененный код может выглядеть так:
```cs
IResult rc = Results.Problem(
    detail: ex?.InnerException?.Message ?? ex?.Message ?? "Panic",
    instance: app.Environment.EnvironmentName,
    title: "ASPA004",
    statusCode: 500
);
```

## Задание 5

**34. Поясните назначение функций (методов интерфейса IRepository и его реализации):**
* `getAllCelebrities()`: возвращает массив всех объектов (знаменитостей) из источника данных (JSON-файла).
* `getCelebrityById(int id)`: выполняет поиск и возвращает объект знаменитости по её уникальному идентификатору. Возвращает `null`, если объект не найден.
* `getCelebritiesBySurname(string Surname)`: возвращает массив объектов, у которых фамилия совпадает с переданным параметром (без учета регистра).
* `getPhotoPathById(int id)`: возвращает путь к фотографии знаменитости по её идентификатору. Возвращает `null`, если объект не найден.
* `addCelebrity(Celebrity celebrity)`: добавляет новый объект в коллекцию, генерируя для него новый уникальный идентификатор. Возвращает сгенерированный идентификатор.
* `delCelebrityById(int id)`: удаляет объект из коллекции по его идентификатору. Возвращает `true`, если объект был успешно удален, и `false`, если объект не был найден.
* `updCelebrityById(int id, Celebrity celebrity)`: обновляет данные существующего объекта по идентификатору. Возвращает `true` в случае успешного обновления.
* `SaveChanges()`: сохраняет все внесенные изменения (результаты работы `addCelebrity`, `delCelebrityById`, `updCelebrityById`) в постоянное хранилище (файл `Celebrities.json`). Возвращает количество измененных записей или `0` при ошибке сохранения.

**35. Поясните назначение следующих объектов и интерфейсов:**
* `IRepository`: Интерфейс (контракт), определяющий набор методов для взаимодействия с уровнем доступа к данным. Использование интерфейса позволяет абстрагироваться от конкретной реализации (например, от того, где физически хранятся данные).
* `Repository`: Класс, реализующий интерфейс `IRepository`. Инкапсулирует логику чтения, изменения и сохранения данных в формате JSON, выполняя роль промежуточного слоя между бизнес-логикой и хранилищем.
* `WebApplicationBuilder`: Паттерн проектирования "Строитель", используемый для конфигурации веб-приложения перед его созданием (добавление сервисов в DI-контейнер, конфигурация логирования, параметров и т.д.).
* `WebApplication`: Представляет настроенное и готовое к запуску веб-приложение ASP.NET Core. Используется для настройки конвейера обработки HTTP-запросов (маршрутизация, добавление middleware).
* `IExceptionHandlerFeature`: Интерфейс, предоставляемый ASP.NET Core внутри обработчика ошибок, который позволяет получить информацию о перехваченном необработанном исключении (свойство `Error` типа `Exception`).
* `HttpContext`: Инкапсулирует всю информацию о текущем HTTP-запросе и HTTP-ответе (заголовки, тело, параметры запроса, статус ответа и т.д.). Доступен во всех обработчиках маршрутов и middleware.

**36. Чем известны Ada Lovelace и Charles Babbage?**
* **Charles Babbage (Чарльз Бэббидж)** — английский математик, изобретатель и философ XIX века. Известен как создатель концепции первого программируемого компьютера (Аналитической машины), которая стала прообразом современной вычислительной техники.
* **Ada Lovelace (Ада Лавлейс)** — английский математик, дочь поэта Джорджа Байрона. Она известна тем, что составила первое в мире описание алгоритма, предназначенного для выполнения на Аналитической машине Бэббиджа (алгоритм вычисления чисел Бернулли). По этой причине её считают первым в мире программистом.
