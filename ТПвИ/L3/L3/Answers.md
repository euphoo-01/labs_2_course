# Ответы на вопросы (Задание 6)

**20. Поясните назначение паттерна Repository.**
Назначение паттерна Repository (Репозиторий) — инкапсуляция логики доступа к данным. Он выступает как промежуточный слой между бизнес-логикой приложения и источником данных, предоставляя объектно-ориентированный интерфейс (обычно в виде коллекций) для выполнения операций над данными (чтение, сохранение, удаление и т.д.). Это изолирует остальной код от специфики конкретного хранилища данных (в данном случае — файловой системы и JSON) и упрощает поддержку приложения.

**21. Для чего в паттерне Repository применяется интерфейс IRepository?**
Интерфейс `IRepository` применяется для реализации принципа инверсии зависимостей (буква 'D' в SOLID). Зависимость бизнес-кода строится на основе абстракции (интерфейса), а не конкретной реализации класса `Repository`. Благодаря этому можно легко подменять реализацию доступа к данным (например, заменить JSON-файл на базу данных SQL) или использовать Mock-объекты для модульного тестирования, не меняя код, который вызывает методы репозитория.

**22. Для чего применяется интерфейс IDisposable?**
Интерфейс `IDisposable` применяется для предоставления механизма детерминированного освобождения неуправляемых ресурсов (например, открытых файлов, дескрипторов операционной системы, сетевых соединений или подключений к БД). Класс, реализующий этот интерфейс, обязан предоставить метод `Dispose()`, в котором и происходит очистка ресурсов до того, как объект будет окончательно уничтожен сборщиком мусора (Garbage Collector).

**23. Поясните работу следующей конструкции языка С# (с ключевым словом using)**
Конструкция `using (IRepository repository = Repository.Create(...)) { ... }` в C# гарантирует, что по завершении блока кода внутри фигурных скобок для созданного объекта `repository` будет автоматически вызван метод `Dispose()`. Это происходит в любом случае — даже если внутри блока `using` возникнет исключение. На уровне компилятора конструкция преобразуется в блок `try { ... } finally { ... }`.

**24. Поясните смысл слов: сериализация и десериализация.**
- **Сериализация** — это процесс преобразования объекта (или структуры данных) из оперативной памяти в формат, который можно сохранить (например, в файл) или передать по сети (строка, поток байтов, JSON, XML).
- **Десериализация** — это обратный процесс воссоздания (восстановления) объекта в оперативной памяти из сохраненного ранее формата (например, чтение строки JSON из файла и преобразование её в массив объектов C# `Celebrity[]`).

**25. Расшифруйте аббревиатуру JSON, поясните смысл.**
**JSON** расшифровывается как **J**ava**S**cript **O**bject **N**otation. Это текстовый формат обмена данными, основанный на подмножестве языка программирования JavaScript. Формат легко читается и пишется людьми, а также легко парсится и генерируется машинами. Он используется для хранения и передачи структурированных данных (представленных в виде пар "ключ-значение" и массивов) между различными компонентами систем, независимо от того, на каком языке программирования они написаны.

**26. ♣ Кто такой Эдсгер Дейкстра? ☺**
Эдсгер Вибе Дейкстра — выдающийся нидерландский ученый, один из пионеров в области компьютерных наук и основоположников структурного программирования. Он известен созданием алгоритма нахождения кратчайшего пути в графе (алгоритм Дейкстры), разработкой концепции семафоров для синхронизации параллельных процессов в многозадачных ОС, а также своим знаменитым письмом "О вреде оператора GOTO" (Go To Statement Considered Harmful), которое навсегда изменило подход к написанию кода.
