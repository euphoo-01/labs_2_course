Анализ показателей финансовой отчетности предприятий. 
В структуру холдинга входят несколько предприятий. 
Работа предприятия оценивается показателями (прибыль, себестоимость и пр.)
Информационные поля: 
Название показателя, Важность показателя, Значение показателя. 
Название предприятия, Банковские реквизиты, Телефон, Контактное лицо,
Дата.

Могут повторяться: Важность показателя, Название предприятия и все что с 
предприятием связано (реквизиты, телефон, лицо)


# 1 NF
> Чтобы таблица соответствовала 1-й нормальной форме (1NF), необходимо, 
чтобы все значения ее полей были неделимыми и не вычисляемыми,
а все записи – уникальными (не должно быть полностью совпадающих строк).
**Показатели:**
- ID записи (Primary Key)
- Название предприятия
- Расчетный счет
- БИК банка
- Телефон
- Контактное лицо
- Название показателя
- Важность показателя
- Значение показателя
- Дата

# 2NF
> Чтобы таблица соответствовала 2-й нормальной форме (2NF), необходимо, 
чтобы она находилась в 1-й нормальной форме и все не ключевые поля 
полностью зависели от ключевого

**Предприятия**:
- ID предприятия (Primary key)
- Название предприятия
- Расчетный счет
- БИК банка
- Телефон
- Контактное лицо

**Показатели**:
- ID записи (Primary key)
- ID предприятия (Foreign key)
- Название показателя
- Важность показателя
- Значение показателя
- Дата

# 3NF
> Для перехода к 3-й нормальной форме (3NF), необходимо обеспечить, 
чтобы все таблицы находились во 2-й нормальной форме и все не ключевые 
поля в таблицах не зависели взаимно друг от друга. 

**Предприятия**:
- ID предприятия (Primary key)
- Название предприятия
- Расчетный счет
- БИК банка
- Телефон
- Контактное лицо

**Типы показателей**:
- ID типа (Primary key)
- Название показателя
- Важность

**Значения показателей**:
- ID значения (Primary key)
- ID предприятия (Foreign key)
- ID типа показателя (Foreign key)
- Значение
- Дата
